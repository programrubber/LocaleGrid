LocaleGrid Product Report

다국어 JSON 편집 업무 개선을 위한 LocaleGrid 플러그인 산출물 보고서

LocaleGrid는 PyCharm/IntelliJ 기반 IDE에서 locales/{locale}/{category}.json 구조의 번역 JSON을 한 화면에서 비교, 수정, 검증, 적용할 수 있도록 지원하는 다국어 에디터 플러그인입니다. 본 문서는 플러그인의 도입 목적, 업무 개선 흐름, 주요 기능, 휴먼 에러 검출 사례, 초기 설정과 배포 정보를 정리합니다.

1. 프로젝트 개요

개발 배경 및 목적

기존 다국어 작업은 언어별 JSON 파일을 각각 열어 key를 수동으로 비교하고 값을 수정하는 방식이었습니다. 파일이 늘어날수록 누락 번역, 중복 key, dot path 충돌, 빈 값 같은 휴먼 에러를 놓치기 쉽고, 저장 후 리뷰나 실제 동작 확인 단계에서 문제가 드러나는 경우가 많았습니다.

LocaleGrid는 동일 category의 locale JSON 파일을 하나의 그리드로 묶어 보여주고, 편집과 검증을 IDE 안에서 처리하도록 하여 다국어 리소스 관리의 반복 작업과 검수 비용을 줄이는 것을 목표로 합니다.

주요 기능 요약 다국어 에디터, 검색/상태 필터, 현황 스크롤맵, key 추가/수정/삭제, 적용 미리보기, JSON 검증, Excel 내보내기를 제공합니다.
기대 효과 locale별 파일 비교 시간을 줄이고, 저장 전 오류를 차단하며, 번역 누락과 중복 key를 빠르게 확인할 수 있습니다.
적용 대상 IntelliJ Platform 2023.3.5 기반, PyCharm 타입 PY, Java 17, sinceBuild 233 이상 환경을 대상으로 합니다.
항목 내용
기본 파일 구조 locales/{locale}/{category}.json, locales/{locale}/{category}_{locale}.json
대표 사용 예 locales/ko/menu.json, locales/en/menu.json, locales/ja/menu.json을 하나의 다국어 에디터에서 비교
지원 편집 값 문자열 value 직접 편집. number, boolean, array, object leaf는 readonly 또는 unsupported 상태로 표시
저장 방식 적용 전 JSON 재생성 및 검증 후, 설정된 JSON indent 기준으로 파일 반영

2. 업무 프로세스 개선

기존 다국어 작업 흐름

  1. locale별 JSON 파일을 직접 탐색한다.
  2. 같은 category 파일을 여러 탭에 열어 key를 수동 비교한다.
  3. 언어별 value를 직접 수정한다.
  4. 빈 값, 누락 key, 중복 key 여부를 사람이 직접 확인한다.
  5. 저장 후 리뷰 또는 동작 확인 단계에서 오류가 발견된다.
문제점: 파일 간 비교 기준이 사람의 눈에 의존하고, 오류 검출 시점이 늦습니다.

LocaleGrid 적용 후 흐름

  1. JSON 파일을 열고 다국어 에디터 탭을 선택한다.
  2. 같은 category의 locale 파일을 자동으로 그룹화한다.
  3. key/value를 그리드에서 비교하고 필요한 값을 수정한다.
  4. 검색, 필터, 현황 스크롤맵으로 변경/경고/에러 위치를 찾는다.
  5. 적용 미리보기와 검증을 거쳐 JSON 파일에 반영한다.
개선점: 비교, 편집, 검증, 적용이 IDE 안의 단일 흐름으로 연결됩니다.

개선된 처리 흐름

1 JSON 파일 열기

현재 파일의 locale과 category를 식별합니다.

2 자동 그룹화

같은 category의 locale 파일을 찾아 key union을 구성합니다.

3 그리드 편집

언어별 value를 한 화면에서 비교하고 수정합니다.

4 검증 및 미리보기

경고와 에러를 표시하고, 적용 전에 변경 요약을 확인할 수 있게 합니다.

5 JSON 반영

설정된 indent 기준으로 locale JSON 파일에 저장합니다.

5. 주요 기능 소개

다국어 에디터 같은 category의 locale JSON을 key 기준으로 병합해 한 화면에서 비교하고 문자열 value를 직접 편집합니다.
검색/상태 필터 key와 value를 검색하고, 추가/편집/삭제/경고/에러 상태별로 행을 필터링합니다.
현황 스크롤맵 에러, 삭제, 추가, 편집, 경고 상태의 행 위치를 세로 마커로 표시하고 클릭/드래그 이동을 지원합니다.
추가/수정/삭제 key 추가, 이름 변경, 값 수정, 범위 선택 삭제, 삭제 취소를 제공합니다. 다중 선택 시 단일 편집은 비활성화됩니다.
검증과 적용 미리보기 적용 전 변경 요약, 생성 파일, 경고/에러 수를 보여주며 에러가 있으면 적용을 차단합니다.
Excel 내보내기 그리드 내용을 Excel 형식으로 내보내 번역 검수나 공유 자료로 활용할 수 있습니다.
기능 사용자가 얻는 효과
상단 검색 입력 좁은 에디터 폭에서도 검색 입력 영역이 잘리지 않도록 반응형 레이아웃을 적용했습니다.
상태 렌더링 행 단위로 추가, 편집, 삭제, 경고, 에러 상태를 구분해 작업 우선순위를 판단할 수 있습니다.
삭제 상태 관리 기존 key는 즉시 제거하지 않고 삭제 상태로 표시해 적용 전 검토와 삭제 취소가 가능합니다.
누락 파일 처리 locale 파일이 없을 때 생성 대상 여부를 적용 전 확인할 수 있습니다.

6. 검출 가능한 휴먼 에러

LocaleGrid는 사람이 여러 locale JSON 파일을 직접 비교하면서 놓치기 쉬운 오류를 상태와 진단 메시지로 표시합니다. 저장을 막아야 하는 항목은 에러로, 확인 후 진행 가능한 항목은 경고로 분리합니다.

검출 항목 예시 표시 수준 조치 방향
중복 key 동일 JSON object 안에 "title" key가 두 번 선언된 경우 에러 중복 항목 중 잘못된 key를 삭제하거나 하나로 병합합니다.
dot path 충돌 device.title과 device.title.short가 동시에 leaf/object로 충돌하는 경우 에러 key 구조를 재정의해 leaf와 object 경로가 겹치지 않게 합니다.
빈 번역 값 "menu.save": ""처럼 value가 비어 있는 경우 경고 번역 값을 입력하거나 의도적으로 비워둔 값인지 확인합니다.
누락 locale 파일 locales/ja/menu.json만 없는 경우 경고 적용 시 생성 여부를 확인하고 필요한 locale 파일을 생성합니다.
JSON parse 실패 쉼표 누락, 따옴표 불일치 등으로 JSON을 읽을 수 없는 경우 에러 원본 JSON 문법을 수정한 뒤 다시 로드합니다.
readonly value 문자열이 아닌 number, boolean, array, object leaf 값 경고 직접 편집 대상인지 확인하고 필요 시 원본 JSON에서 구조를 조정합니다.
적용 전 오류 차단 에러 진단이 남아 있는데 적용 버튼을 누른 경우 차단 에러를 먼저 해소한 뒤 적용합니다.

7. 장비별 검출 사례

아래 사례는 실제 운영 장비명을 특정하지 않은 보고용 예시입니다. 각 장비/프로젝트 환경에서 발생할 수 있는 대표적인 key 값 중복, 빈 값, dot path 충돌을 LocaleGrid가 어떻게 보여주고 조치 흐름으로 연결하는지 설명합니다.

장비 A: key 값 중복

중복 key
발생 원인

locales/ko/menu.json에서 동일 object 안에 "save" key가 두 번 선언되었습니다.

LocaleGrid 검출 방식

JSON 로드 시 duplicated JSON key를 에러로 진단하고, 해당 key 행을 에러 상태로 표시합니다.

사용자 조치

중복 항목 중 잘못된 항목을 삭제 상태로 변경하거나 올바른 key로 이름을 변경합니다.

기대 효과

같은 object 안에 중복 key가 있으면 마지막 값만 남을 수 있으므로, 번역이 의도와 다르게 적용되는 문제를 사전에 줄입니다.

장비 B: 빈 값 검출

빈 번역
발생 원인

locales/en/menu.json의 menu.delete value가 빈 문자열로 남아 있습니다.

LocaleGrid 검출 방식

빈 값 진단을 경고로 표시하고, 경고 필터와 스크롤맵에서 해당 위치를 보여줍니다.

사용자 조치

영문 번역 값을 입력하거나, 의도적으로 비워둔 값이면 적용 미리보기에서 경고를 확인하고 진행합니다.

기대 효과

화면에 빈 라벨이 노출되거나 기본 locale만 표시되는 문제를 조기 발견합니다.

장비 C: dot path 충돌

구조 충돌
발생 원인

locales/ja/menu.json에 device.title과 device.title.short가 함께 존재합니다.

LocaleGrid 검출 방식

JSON 재생성 검증에서 leaf와 object 경로가 충돌하는 key를 에러로 표시하고 적용을 차단합니다.

사용자 조치

device.title을 device.title.text처럼 구조에 맞는 key로 변경합니다.

기대 효과

저장 시 JSON 구조가 깨지거나 일부 번역이 손실되는 위험을 줄입니다.

8. 초기 설정 및 사용 가이드

설치 방법

  1. PyCharm 또는 IntelliJ에서 Settings > Plugins로 이동합니다.
  2. 기어 아이콘을 선택하고 Install Plugin from Disk를 실행합니다.
  3. release/LocaleGrid-0.7.0.zip 파일을 선택해 설치합니다.
  4. IDE를 재시작한 뒤 JSON 파일을 열어 다국어 에디터 탭을 확인합니다.

프로젝트 설정

설정 항목 기본값/예시 설명
locale 루트 경로 locales locale 디렉터리가 위치한 프로젝트 상대 경로입니다.
locale 표시 순서 ko,en,ja,vi 그리드에서 locale 열을 표시할 순서입니다. 비워두면 자동 감지를 사용합니다.
예외 key __section__ root-level entry로 보존해야 하는 예외 key입니다.
JSON 들여쓰기 2 적용 시 JSON 파일에 사용할 indent 크기입니다.

기본 사용 흐름

1 파일 열기
